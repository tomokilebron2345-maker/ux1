📚 6週目

UXリサーチ + ユーザー理解

データと共感によるユーザー理解を深める

🎯 この週のミッション

定量調査と定性調査、行動データ分析を通じてユーザーを多角的に理解し、ペルソナやカスタマージャーニーマップなどのモデルを作成することで、ユーザー中心のUXデザインの基礎を構築します。

🔍 UXリサーチとは

UXリサーチは、ユーザーのニーズ、行動、感情を理解するための系統的な調査です。定量的・定性的なデータを組み合わせることで、より正確な洞察が得られます。

UXリサーチの位置づけ

定義 「ユーザーの行動、ニーズ、価値観を理解するための調査研究。定量的データ（アンケートなど）と定性的データ（インタビューなど）を組み合わせて行う」

UXリサーチにおいて重要な視点・考え方

  • ユーザーの声を聴く：何を言っているか、何を考えているか、何を感じているかを理解
  • 観察する：ユーザーが実際に何をしているかを見る
  • 行動を記録する：デジタル行動ログなどの客観的データを分析
  • 複合的アプローチ：定量と定性の両方を組み合わせる
  • 継続的改善：リサーチは一度ではなく、継続的に実施する

覚えやすいコツ

「UXリサーチ = 聴く + 観察する + 記録する」と覚えましょう。複数の調査方法を組み合わせることで、ユーザーの全体像が見えてきます。

📊 定量調査

定量調査は、ユーザーの思考や行動を「数字」で把握する手法です。統計的に信頼度の高い結果が得られます。

定量調査とは

定義 「ユーザーの思考や行動を量的に把握する調査。アンケートやA/Bテストなどにより、統計的に分析可能なデータを収集する」

定量調査の主要な手法

アンケート調査 大量のユーザーに対して、統一された質問を行い、回答を集計・分析する。サンプルサイズが大きいほど信頼度が高い。
クロス集計 複数の質問項目の組み合わせを分析し、ユーザーセグメント間の違いを明らかにする。例：年代別の利用率の違い。
サンプリング 全数調査が難しい場合、代表的なサンプルを抽出して調査する。サンプルの選択方法が重要。

定量調査の設計方法

  • 自由回答法：回答者が自由に記述する形式。深い情報が得られるが、集計が困難
  • SA（Single Answer）：1つの選択肢のみ選ぶ形式
  • MA（Multiple Answer）：複数の選択肢を選ぶ形式
  • プリコード法：選択肢をあらかじめ決めておく方法

定量調査において重要な視点

  • 設問の中立性：回答を誘導しない質問設計
  • サンプル数：統計的信頼性を確保するのに十分なサンプルサイズ
  • 回答率：回答率が低いと、バイアスが生じる可能性
  • 分析の視点：相関関係と因果関係を区別する

💬 定性調査

定性調査は、ユーザーの思考や行動を「言葉や行動」で把握する手法です。深い洞察が得られます。

定性調査とは

定義 「ユーザーの思考や行動を定性的に把握する調査。インタビューやフィールドワークなどにより、ユーザーの本当のニーズや行動パターンを発見する」

定性調査の主要な手法

インタビュー ユーザーと直接対話し、詳細な情報を得る。半構造化インタビューが一般的。
グループインタビュー 複数のユーザーを集めて、グループディスカッションから集団力学による洞察を得る。
ユーザー観察（フィールドワーク） 実際の使用環境でユーザーの行動を観察する。文脈に基づくニーズが発見できる。
エスノグラフィー 長期間にわたってユーザーの生活や行動を詳細に観察・記録する。文化人類学的なアプローチ。
ダイアリー法 ユーザーに日記形式で行動や感情を記録してもらう。時系列での変化を追跡できる。

定性調査の設計方法

  • リクルーティング：適切なユーザーペルソナを特定し、参加者を募集する
  • インタビューガイド：質問項目を事前に準備するが、柔軟に対応
  • KJ法：インタビュー結果を分類・整理し、パターンを発見する

覚えやすいコツ

「定性調査は『なぜ？』に答える」と覚えましょう。数字にはできない、ユーザーの深い動機や背景にある課題を発見することが、定性調査の価値です。

📈 行動データ分析

行動データ分析は、ユーザーの実際のデジタル行動ログから、客観的なパターンを発見します。

行動データ分析とは

定義 「ユーザーIDに紐づく行動データ（ページビュー、クリック、滞在時間など）を分析し、ユーザーの一連の行動の流れを把握する手法」

行動データ分析の主要な手法

行動フロー ユーザーがどのページからどのページに移動するかの流れを可視化する。ボトルネック発見に有効。
シーケンス分析 ユーザーの行動パターンを一連のシーケンスとして分析する。共通のパターンを発見できる。
行動ギャップ ユーザーの「言ったこと」と「実際にしたこと」の乖離を発見する。本当のニーズ把握に重要。
ペインポイント ユーザーが困っている箇所。直帰率が高いページや、エラー発生箇所など。
ゲインポイント ユーザーが満足している箇所。滞在時間が長いページや、コンバージョン箇所など。

行動データをUXデザインに活用する手法

  • A/Bテスト：2つの異なるデザイン案を比較し、どちらがユーザーの行動に影響するか測定
  • ヒートマップ：ユーザーがどこをクリックしているかを可視化
  • ファネル分析：各ステップでのコンバージョン率を分析

👥 ユーザーモデリング

ユーザーモデリングは、リサーチ結果から得られたユーザーの情報を体系的に整理し、設計の指針とします。

主要なモデリング手法

ペルソナ ターゲットユーザーを代表するの架空のキャラクター。年齢、職業、興味、ニーズなどを詳細に記述。設計判断の指針となる。
カスタマージャーニーマップ ユーザーが製品・サービスと関わる一連のプロセスを時系列で可視化。各段階でのユーザーの感情や課題を示す。
UXカーブ ユーザーの体験の質を時系列で表現したグラフ。満足度が高い時点と低い時点を可視化し、改善機会を発見。
タスク分析 ユーザーがタスクを完了するために必要な行動ステップを詳細に分解する。各ステップでの認知負荷を把握。
KA法（Key Activities） タスク分析と同様に、ユーザーの活動を細分化して、どこで支援が必要かを明確にする。
エンパシーマップ ペルソナが「見ているもの」「聞いているもの」「話しているもの」「考えていること」「感じていることなど」を可視化。深い共感を生み出す。
ジョブ理論 ユーザーが製品やサービスを使用する「目的」（ジョブ）に焦点を当てる。製品の機能ではなく、ユーザーが解決したい課題に基づくセグメンテーション。

🎯 ユーザー要求定義

ユーザー要求定義は、ユーザーモデリングの結果から、設計の指針となる要件を抽出します。

ユーザー要求の種類

現在の利用状況の把握（AS-IS分析） ユーザーが現在、どのような方法で課題に対処しているか、現在の体験はどうか。既存の行動パターンを理解。
理想の利用状況の想定（TO-BE分析） ユーザーが理想とする体験はどのようなものか。制約を考慮せず、理想の状態を描く。

理想の利用状況を発想するプロセス

  • バリュープロポジション：「ユーザーが達成したい目標」と「私たちの提供する価値」の交点を見つける
  • ユーザーストーリーマッピング：「ユーザーとして、私は〇〇したい。なぜなら△△だから」という形で要求を言語化
  • ワークショップ：デザイナー、PM、ステークホルダーで、理想の体験を共創する

覚えやすいコツ

「現在（AS-IS）→ 理想（TO-BE）→ ギャップを埋める設計」と覚えましょう。ユーザー要求定義は、ユーザーの現在と理想の乖離を理解し、その間を埋めるための基礎となります。

🔗 リサーチから設計への流れ

段階 活動 成果物
1. 定量調査 アンケート、統計分析 数字による全体的なトレンド
2. 定性調査 インタビュー、観察 ユーザーの本当のニーズ、背景
3. 行動データ分析 ログ分析、A/Bテスト 実際の行動パターン、ボトルネック
4. ユーザーモデリング ペルソナ作成、ジャーニーマップ作成 ユーザーの代表像、体験の流れ
5. ユーザー要求定義 AS-IS/TO-BE分析、ユーザーストーリー 設計の要件、ユーザーストーリー
6. 設計実行 ワイヤーフレーム、プロトタイプ作成 実装可能な設計案

  • ペルソナの役割と作成方法を説明できる

📌 W6のまとめ（5つの要点）

  1. 複合的リサーチアプローチ — 定量と定性、行動データを組み合わせることで、ユーザーの全体像が見える
  2. 定量調査は「何か、どのくらいか」 — 統計的な信頼度の高い結果を得られるが、背景や理由は不明
  3. 定性調査は「なぜか」 — 深いニーズと動機を発見できるが、サンプル数は限定的
  4. 行動データは客観的真実 — ユーザーが「言ったこと」ではなく、「実際にしたこと」を明らかにする
  5. ユーザーモデリングが設計の指針 — リサーチ結果をペルソナやジャーニーマップなどのモデルに落とし込むことで、全員が共通のユーザー理解を持つことができる
