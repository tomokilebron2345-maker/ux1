🤔UXの第一印象

UX(User Experience) を日本語に訳すと：「ユーザー体験」

でも、これだけじゃ曖昧だから、世界中の専門家が定義しています。

📚 3つの有名な定義

1
🌍 ISO（国際標準化機構）
製品・サービス・システムの使用を通じてユーザーが知覚・獲得するすべての感覚・感情・認識

💡 つまり：「全体的な印象」
2
🏆 Nielsen & Norman（UXの大家）
ユーザーが製品やサービスとのやり取りを通じて形成する感覚的な評価のすべて

💡 つまり：「すべての感覚」
3
🎯 UXPA（UX専門家協会）
ビジネス目標を達成しながらユーザーニーズを満たす体験の設計と実装

💡 つまり：「ビジネス × ユーザー」
💡 覚えやすいコツ：「3つの定義の違い」
ISO = 「すべて」（感覚・感情・認識すべてをカバー）
Nielsen & Norman = 「感覚的な評価」（より実感的な定義）
UXPA = 「ビジネス視点」（実務的な定義）

🎯 覚え方：「グローバル → 実感的 → ビジネス」と段階的に実務寄りになっていく！

❓UIとUXの違い

よく混同されるUIとUX。スマホで食べ物を注文するアプリで説明します。

🖌️
【UI = 見た目・操作部分】
✅ ボタンの色は？
✅ 文字のサイズは？
✅ 写真はどこに配置？
✅ ボタンをタップしやすいか？

💡 目に見える「インターフェース」

⬇️ ⬇️ ⬇️
💭
【UX = 全体の体験】
✅ 欲しい商品は見つかった？
✅ 決済は簡単？
✅ 商品は時間通りに届いた？
✅ また使いたいと思った？

💡 心の中の「体験全体」

⭐ 覚えやすいコツ：「UI vs UX」
UI = User「Interface」 → インターフェース（目に見える部分）
UX = User「Experience」 → エクスペリエンス（心の体験）

🎯 覚え方：「UI は見た目、UX は気持ち」
または「UI は楽器、UX は音楽」と覚えると良いです！

🤯DX・CX・EXとの違い

UXと似た言葉がいっぱいあります。それぞれの違いを整理しましょう。

言葉 対象者 意味
UX
(ユーザー体験)		 ユーザー 製品・サービスの利用体験
CX
(カスタマー体験)		 顧客 企業との全ての接点における体験
（UXより広い）
DX
(デジタル化)		 企業全体 デジタル化によりビジネスを
根本的に変革すること
EX
(従業員体験)		 従業員 会社での働く体験
🎯 覚えやすいコツ：「4つのXの関係図」
外側から内側へ拡大する関係

🎯 DX（企業全体）
↓
👥 CX（全接点）
↓
📱 UX（製品体験） + 👨‍💼 EX（従業員体験）

💡 ニーモニック：「D C UE」を思い出す時は
「Digital Changes User and Employee」

🔍 具体例：Amazonで本を買うとき

📱 UX ： アプリで本を検索 → 購入 → レビュー
（製品としてのアプリの体験）
🛍️ CX ： 検索 → 購入 → 配送 → 到着 → レビュー
（顧客全体の体験）
💻 DX ： Amazonがデータで企業全体を変革
（テクノロジーによる経営変革）
👥 EX ： Amazon社員の働く環境と体験
（従業員の体験）

🚀なぜ今、UXが重視されるのか？

📊 昔と今でビジネスが変わった！

【昔の世界】2000年代初期
📺 テレビで広告 → 📦 店舗で購入 → ✨ 完結

「１つのチャネル」

👉 UXはあまり重要じゃなかった
　　「とにかく売る」が目的だった
⬇️
テクノロジーで変わった
【今の世界】2020年代
📱 SNS → 🌐 ウェブ → 📲 アプリ → 🏪 店舗 → 📞 サポート → 💬 コミュニティ

「複数のチャネルが連動」

👉 「毎回、いい体験」が必須！
　　 一つでも悪いと別のサービスに乗り換える
💡 覚えやすいコツ：「チャネル数の増加」
昔：1つのチャネル = 1回きりの関係
今：複数チャネル = 継続的な関係

🎯 覚え方：「1回の満足 vs 毎回の満足」
昔は「買ってくれればOK」でしたが、
今は「ずっと使ってくれることが成功」に変わった！

💰ビジネスモデルの大転換

❌ 売り切り型
商品を買ったら終わり
⬇️
テクノロジーで変わった
✅ サブスク型・継続型
毎月使い続けてもらうために
「常にいい体験」が必要！

📊 数字で見る変化

【従来】
売上 = 1回の購入 × 顧客数
例：1000円 × 100人 = 10万円（終わり）
【現在】
売上 = 月額料金 × 継続月数 × 顧客数
例：1000円 × 12ヶ月 × 100人 = 120万円
💡 覚えやすいコツ：「12倍の魔法」
売り切り型：100円 × 100人 = 1万円
サブスク型：100円 × 100人 × 12ヶ月 = 12万円

🎯 覚え方：「サブスクは12倍の可能性！」
だから「毎回いい体験」が超重要なんです。

💻テクノロジーが変えたこと

📊 ビッグデータが使えるようになった

❌
昔：「なんとなく」で設計
👉 失敗する確率が高かった
✅
今：「データ」で設計
💾 ユーザーの行動を記録
📊 何が好きか分析
📈 改善効果を測定
👉 成功確率がアップ！

🤖 AI・機械学習の活用

例：Netflixの推奨機能
📺 「あなたにおすすめ」
← これはAIがUXを自動最適化している！
あなたの視聴履歴 → AI分析 → 最適な作品提案
💡 覚えやすいコツ：「データドリブン = UXドリブン」
データに基づいた意思決定 = より良いUXが生まれる

🎯 覚え方：「データはUXの羅針盤」
データがあれば、ユーザーの本当のニーズが見える！

📚W1の重要ワード集

これら14個のワードは、W2以降でも何度も出てきます。

UX (User Experience) - ユーザー体験
UI (User Interface) - ユーザーインターフェース
DX (Digital Transformation) - デジタル化による変革
CX (Customer Experience) - カスタマー体験
EX (Employee Experience) - 従業員体験
UXのハニカム構造 - 7つの要素
ユーザビリティ - 使いやすさ
アクセシビリティ - 誰でも使える
OMO - Online Merges with Offline
D2C - Direct to Consumer
プラットフォームビジネス - 参加者が相互作用
ビッグデータ - 大量のデータ
データドリブン - データに基づいた意思決定
サブスクリプション型 - 継続利用モデル
💡 覚えやすいコツ：「グループ化して覚える」
グループ1：体験系 → UX / UI / CX / EX
グループ2：ビジネス系 → OMO / D2C / プラットフォーム
グループ3：データ系 → ビッグデータ / データドリブン
グループ4：モデル系 → サブスク / DX / ハニカム

🎯 覚え方：4つのグループに分けると覚えやすい！
